Why I Build These

Building AI Tools for Real Organizations

Working inside public health programs taught me to think in workflows, constraints, and handoffs: who uses this system, what breaks when they leave, how do you make it survive the person who built it. Building AI tools turned out to be a natural extension of that same thinking.

Each of these systems was scoped, designed, and shipped solo. They range from a Claude-powered voice intake system built for community health caseworkers, to an AI research study design platform that grew out of my dissertation, to an open-source model router that makes LLM access affordable for resource-constrained teams.

What ties them together: I build things that solve real problems for specific users, I design them so non-technical people can use them, and I think about what happens after I'm gone.

AI Agent · Voice Interface · Healthcare Intake

GRACE — Guided Responsive AI Conversation Engine

A prototype voice-enabled AI intake agent built for Healthy Start SC caseworkers — replacing a static paper-based intake form with a conversational agent that conducts the full federal Pregnancy Risk Assessment interview, including clinical safety screening, in natural spoken dialogue.

🎙️ Voice input → Claude API conversation logic → EPDS / IPV safety checks → 🔊 ElevenLabs voice reply → REDCap-ready output
  • Used Claude API as the conversational engine, ElevenLabs TTS (Sarah voice) for natural voice synthesis, and the Web Speech API for live microphone input — creating a full two-way voice interview experience
  • Designed EPDS depression screening and IPV safety-flagging logic directly into the AI system prompt, embedding clinical safety protocols at the model level with staff-only field protections
  • Built JWT-based staff authentication, a participant-facing voice interview interface, and automated document generation producing REDCap-ready data entry sheets from each completed interview
  • Covers the full Healthy Start SC Pregnancy Risk Assessment form — translating a 30+ field clinical intake into a guided, adaptive conversation

AI Platform · Full-Stack · Research Tools

TTEngine — AI-Powered Research Study Design

A full-stack AI application that helps researchers design rigorous observational studies from real-world data. Born out of my own dissertation work, where I realized the methodology I was using (target trial emulation) was powerful but inaccessible to most researchers without deep causal inference training. TTEngine translates that complexity into guided, step-by-step protocols.

Research question input → Claude API protocol generation → Interactive study design → Exportable protocol document
  • Scoped, architected, and shipped independently from concept through working product, including FastAPI backend, React frontend, and Claude API-driven protocol generation
  • Designed for non-specialist researchers: the system walks users through study design decisions that would normally require a causal inference methodologist, producing structured protocols they can follow and share
  • Related work accepted at SER 2026 (Society for Epidemiologic Research), validating the underlying methodology
  • Addresses a real gap: most research teams lack access to target trial emulation expertise, and existing tools assume deep statistical knowledge

AI Infrastructure · Open Source · Cost Optimization

Project Coffee — AI Model Router & Cost Optimizer

An open-source personal AI workstation that intelligently routes prompts to the most cost-efficient model based on task complexity. Built around a playful coffee-shop metaphor: models are "beans," costs are tracked in a "tips jar," and any operation above a cost threshold requires explicit user approval before running.

User prompt → Task complexity analysis → Model selection (cost-optimized) → Approval gate (if expensive) → Response
  • Built and open-sourced on GitHub as a tool that solves a real adoption barrier: organizations that want to use powerful AI models but need cost predictability and transparency
  • Designed the routing logic to balance quality and cost, particularly relevant for resource-constrained organizations (nonprofits, public agencies, small teams) exploring LLM adoption for the first time
  • Features a collapsible animated coffee-counter UI with a barista character that reacts to backend pipeline states, streaming response display, and a prompt interface with model-alias "beans" and cost tracking
  • Explicit cost-approval threshold prevents surprise spending, giving users control over when to escalate to more expensive models
Open-source on GitHub

OutreachIQ — Autonomous Networking Agent

An end-to-end autonomous AI agent that discovers job opportunities, analyzes requirements, scores fit against a candidate profile using LLM-based evaluation, and drafts personalized outreach emails, all without manual intervention between steps.

Job discovery → LLM fit scoring → Personalized email draft → Gmail send → PostgreSQL tracking
  • Integrated five external services (web search, Gmail, LLM scoring, contact discovery, PostgreSQL tracking) into a single LangGraph pipeline that maintains state across the full workflow
  • Automated personalized outreach workflows, generating tailored networking emails, cover letters, and faculty introductions based on each job description, research interests, and institutional priorities
  • Built as a self-directed tool to scale my own job search, demonstrating autonomous multi-step agent architecture with real-world integration
Actively used in my own job search

Automated Marketing Video Pipeline

A fully autonomous content pipeline that researches trending topics, writes its own scripts, generates voiceovers, assembles finished video, and publishes across platforms — running on a daily schedule with no manual editing step.

🔎 Trend detection → Claude API script → ElevenLabs voiceover → Shotstack assembly → AssemblyAI captions → 📺 YouTube / TikTok publish
  • Built an automated pipeline that researches trending topics, generates AI-written scripts via Claude API, synthesizes voiceovers with ElevenLabs, and assembles ~7-minute stock-footage videos with auto-captions using Shotstack and Pexels
  • Integrated seven external APIs — trend detection, LLM scripting, TTS, video rendering, transcription (AssemblyAI), and YouTube publishing — into a single modular pipeline with full logging and error handling
  • Implemented YouTube OAuth2 refresh-token authentication for unattended daily uploads, automated affiliate-link substitution, and a TikTok export module producing platform-optimized 58-second cuts with overlay branding
  • Runs end-to-end on a daily schedule — from topic discovery to multi-platform publishing — with no manual editing step
